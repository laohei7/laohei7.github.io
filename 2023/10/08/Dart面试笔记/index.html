var、dynamic、Object

var：定义的类型不可变，变量的类型在初始化时根据值的类型进行推断，并且在编译时进行类型检查。

dynamic：定义的类型可变，变量的类型在运行时可以更改，但在编译时不进行类型检查，因此需要小心使用以避免类型错误。

Object：它可以引用任何Dart对象，但在不进行类型转换的情况下不能访问对象的成员。

const、final

const：

  • 必须初始化，只能赋值一次（编译时期赋值）；
  • const关键字即可修饰变量也可用来修饰常量构造函数；
  • 当const修饰类的构造函数时，它要求该类的所有成员都必须是final的；
  • const修饰的List集合任意索引不可修改；
  • const不可以修饰实例变量，访问类中const修饰的变量需要static修饰。

final：

  • 必须初始化，只能赋值一次（运行时期赋值）；
  • final可修饰实例变量；
  • final修饰的List集合任意索引可修改；
  • final 只可用来修饰变量。

??、??=

?? ——> 左边如果为空返回右边的值，否则不处理。

??= ——> 左边如果为空把B的值赋值给A。

使用 get 和 set 方法来实现属性的读取和设置。这可以帮助你控制属性的访问，例如对属性的读取和写入进行自定义操作或添加验证逻辑。

..：返回的相当于是this，当前的对象引用。

空安全

使用?修饰类型，即可声明变量为可空类型。?.和!空安全操作符。?.在变量为空情况下不会执行调用的方法，但不阻塞后面逻辑运行；!在变量为空时抛出异常。

线程模型

Dart为单线程模型。

Main > MicroTask > EventQueue

先开启app执行入口函数main()，执行完成之后，消息机制启动，先是会按照先进先出的顺序逐个执行微任务队列中的任务microtask，事件任务eventtask执行完毕后便会退出，但是，在事件任务执行的过程中也可以插入新的微任务和事件任务，在这种情况下，整个线程的执行过程便是一直在循环，不会退出，而Flutter中，主线程的执行过程正是如此，永不终止。

在事件循环中，当某个任务发生异常并没有被捕获时，程序并不会退出，而直接导致的结果是当前任务的后续代码就不会被执行了，也就是说一个任务中的异常是不会影响其它任务执行的。

介绍

IgnorePointer是 Flutter中的一个小部件，它用于包装其他小部件，并可以使这些小部件对用户输入（例如触摸事件）变得不可用。如：当页面处于登陆中时，登陆按钮点击无效。

【每日一个小组件】StatefulBuilder

 2023年10月9日

介绍

StatefulBuilder是 Flutter 中的一个小部件，它允许你在不重建整个父小部件的情况下更新其内部状态。通常情况下，当需要在构建方法之外更新小部件的状态时，你会使用StatefulBuilder。

StatefulBuilder接受一个回调函数作为其子部件，并将父小部件的BuildContext和一个StateSetter参数传递给该回调函数。StateSetter允许你在回调中更新小部件的状态。
